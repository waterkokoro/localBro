## 简介
本文件面向 LocalBro 的数据流架构，系统性梳理从用户操作到文件系统修改的完整数据通路，覆盖前端事件驱动与状态管理、Tauri 事件系统、并发数据流（目录扫描队列与后台任务）、以及数据一致性保障（状态同步、缓存更新与错误恢复）。文档同时提供面向初学者的概念解释与面向高级开发者的性能与并发细节。

## 项目结构
LocalBro 采用前端 React + Zustand + Tauri 的分层架构：前端负责 UI 与交互、状态管理；Tauri 负责命令桥接与后端执行；核心模块在 Rust 中实现文件系统操作、集合与尺寸索引等能力。

- 应用入口与根组件
  - 入口挂载 React 根节点，渲染应用根组件。
  - 根组件负责初始化、注册预览适配器与皮肤、监听尺寸更新事件、启动目录大小扫描队列、绑定快捷键等。

  - 将前端调用映射为后端命令，统一参数与返回值格式，屏蔽前后端差异。

## 架构总览
LocalBro 的数据流遵循“前端事件 → 状态更新 → 命令调用 → 后端执行 → 事件回传 → 状态更新 → UI 渲染”的闭环。Tauri 事件系统用于后端向前端推送异步结果（如目录扫描完成），前端通过订阅事件更新状态并触发 UI 变化。

```mermaid
sequenceDiagram
participant U as "用户"
participant UI as "前端组件<br/>App/FileList/Sidebar"
participant ST as "状态管理<br/>Zustand"
participant API as "API 封装<br/>invoke"
participant CMD as "命令处理器<br/>commands.rs"
participant FS as "文件系统<br/>fs_ops.rs"
participant SI as "尺寸索引<br/>size_index.rs"
U->>UI : 操作点击/键盘/导航
UI->>ST : 触发动作navigate/setSort/openPreview...
ST->>API : 发起命令如 listDir/requestDirSize
API->>CMD : tauri : : invoke(...)
CMD->>FS : 执行文件系统操作
CMD->>SI : 启动后台扫描或查询缓存
SI-->>CMD : 返回缓存/触发 size-updated 事件
CMD-->>API : 返回结果
API-->>ST : Promise 解析
ST-->>UI : 更新状态并触发重渲染
CMD-->>UI : emit("size-updated", payload)
UI->>ST : 监听事件并更新 dirSizes
ST-->>UI : 状态变化驱动 UI 刷新
```

### 组件一：应用根组件与事件驱动
- 初始化流程
  - 启动时加载内置预览适配器与皮肤，随后初始化状态（获取用户主目录、默认快捷方式、卷与集合）。
  - 导航至用户主目录，准备初始列表。
- 事件驱动
  - 订阅后端发出的 size-updated 事件，收到后更新目录大小缓存，触发 UI 重新计算显示。
- 并发扫描队列
  - 基于当前条目与未缓存目录，维护最多 4 个并发的请求，逐个出队并调用后端请求扫描，忽略单点错误，完成后更新缓存。

```mermaid

Queue --> Concurrency{"并发数 < 4 ?"}
Concurrency --> |是| Next["取出下一个目录"]
Next --> Request["requestDirSize(path)"]
Request --> Done{"成功?"}
Done --> |是| Update["setDirSize(path, bytes)"]
Done --> |否| Ignore["忽略错误"]
Update --> Concurrency
Ignore --> Concurrency
Concurrency --> |否| Wait["等待活跃任务完成"] --> Concurrency
```

## 性能考量
- 并发扫描队列
  - 通过固定并发上限（默认 4）平衡吞吐与资源占用；利用 in-flight 避免重复扫描。
- 缓存策略
  - 目录大小缓存减少重复 IO；失效接口允许手动清理过期数据。
- UI 渲染优化
  - useMemo 仅在依赖变化时重排；目录大小按需计算，避免全量重算。
- I/O 降级

## 故障排查指南
- 列表为空或加载失败
  - 检查 error 状态与 API 返回；确认路径存在且具有访问权限。
- 目录大小不更新
  - 确认 size-updated 事件已监听；检查缓存是否命中；必要时调用失效接口后重试。
- 删除/移动异常
  - 检查是否存在同名目标；跨设备移动会触发降级路径；查看后端日志定位具体错误。
- 预览无法打开
  - 确认预览路径有效且非目录；检查预览适配器注册情况。
