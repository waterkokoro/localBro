# v0.1 — MVP (Released)

Goal: a fast, native file browser with the quality-of-life features
the stock Finder/Explorer misses. macOS first, Windows/Linux on the
same Tauri build.

## Features

### Browsing
- Sidebar: Home / Desktop / Documents / Downloads / Pictures / Music
  / Videos shortcuts, and mounted Volumes.
- Breadcrumb address bar with double-click to edit.
- Back / Forward / Up / Refresh.
- Three view modes: List, Grid, Details.
- Sort by Name / Size / Modified / Kind.

### File operations (Rust core)
- `list_dir`, `stat`, `rename`, `move_to_trash`, `delete_forever`,
  `copy_path`, `move_path`, `create_directory`, `create_file`,
  `reveal_in_native`.
- All paths returned to JS are normalised; errors carry structured
  codes (`FsError`).

### Real-time directory sizes (Task 3)
- Background scan queue (4-way concurrent) pushes `size-updated`
  events; visible list picks them up live.
- Results cached per path; LRU eviction when memory pressure grows.
- Invalidation on user-visible mutation (rename / delete / move).

### Preview (Task 7)
- Space = QuickLook-style modal.
- Built-in adapters: image, text (UTF-8 snippet), PDF (object embed),
  audio / video (HTML5 media). Plugin adapters come in v0.3.

### Hidden files
- `showHidden` toggle in the toolbar is persisted with the OS-level
  hidden attribute (not the naive "starts with dot" rule).

## Non-goals (v0.1)
- Multi-tab browsing.
- Split views.
- FTP / SMB / cloud mounts.
- Search indexing.

## Design notes
- Tauri 2 (Rust backend) + React 19 + zustand store.
- `grid-template-areas` layout so new panes (AI, preview) can dock
  without reshuffling existing styles.
- CSS tokens only — no raw colours in components. This is what makes
  the Skin system in v0.2 trivial.

## Decision log
- *2026-03* — Tauri 2 over Electron: ~20× smaller binary, native FS
  access without IPC marshalling of large payloads.
- *2026-03* — zustand over Redux: three-line store file, no reducers
  for a mostly-local state model.
